## 1. 개요

리눅스에서 **프로세스 상태를 확인하고 관리**할 때 자주 사용하는 명령어는 다음 네 가지다.

- `top`  : 실시간 프로세스/시스템 자원 모니터링
- `ps`   : 현재/과거 시점의 프로세스 목록 조회 (스냅샷)
- `jobs` : 현재 셸에서 실행 중인 백그라운드 작업 관리
- `kill` : 프로세스/잡에 시그널(종료, 일시정지 등) 보내기

---

## 2. 명령어 한눈에 보기

| 명령어 | 주요 역할                         | 대표 사용 예시                 |
|--------|-----------------------------------|--------------------------------|
| `top`  | CPU, 메모리, 프로세스 상태 실시간 조회 | `top`, `top -u 사용자명`        |
| `ps`   | 현재 실행 중인 프로세스 목록 출력 | `ps aux`, `ps -ef \| grep ssh` |
| `jobs` | 현재 셸의 백그라운드 작업 목록 조회 | `jobs`, `fg %1`, `bg %2`       |
| `kill` | 프로세스/잡에 시그널 전송 (종료 등) | `kill 1234`, `kill -9 1234`    |

## 3. `top` – 실시간 시스템/프로세스 모니터링

`top`은 **실시간으로 CPU/메모리 사용량과 프로세스 상태**를 보여주는 모니터링 도구다.  
서버 운영할 때 가장 먼저 치는 명령어 중 하나.

### 3.1 기본 사용법

top

`상단`: 시스템 전체 정보 (uptime, load average, 전체/사용 메모리 등)

`하단`: 프로세스별 정보 (PID, USER, %CPU, %MEM, TIME+, COMMAND 등)

3.2 자주 쓰는 옵션
``` bash
top            # 기본 실행
top -u USER    # 특정 사용자(USER)가 실행한 프로세스만 보기
top -p PID     # 특정 PID만 모니터링
top -n 5       # 5번 갱신 후 자동 종료
```
### 3.3 실행 중 단축키
top 실행 화면에서:
P : CPU 사용량 기준 정렬

M : 메모리 사용량 기준 정렬

k : 특정 PID 프로세스를 kill (PID 입력 후 시그널 선택)

q : 종료

## 4. ps – 프로세스 목록 스냅샷 조회
ps는 현재 시점의 프로세스 상태를 한 번 출력하는 명령어다.
스크립트에서 특정 프로세스를 찾거나, grep과 함께 자주 사용한다.

### 4.1 기본 사용법
``` bash
ps        # 현재 셸에서 실행 중인 프로세스만
ps aux    # BSD 스타일 전체 프로세스
ps -ef    # System V 스타일 전체 프로세스
```

### 4.2 자주 쓰는 조합 & 옵션
명령어	설명
``` bash
ps aux	# 모든 프로세스를 자세히 출력
ps aux | head	# 상위 몇 개만 빠르게 확인
ps aux | grep <이름> # 특정 이름이 포함된 프로세스 검색
ps -ef	#  전체 프로세스를 풀 포맷으로 출력
ps -u USER	# 특정 사용자 프로세스만 출력
ps -p PID	# 특정 PID의 프로세스 정보 출력
```

예시 1) nginx 관련 프로세스 찾기

``` bash
ps aux | grep nginx
```

예시 2) PID와 부모 PID(PPID) 보기
``` bash
ps -ef | head
```

## 5. jobs – 현재 셸의 백그라운드 작업 관리
jobs는 현재 셸에서 실행한 백그라운드 작업 목록을 보여준다.
중요한 포인트: 현재 셸 세션 기준이라는 것. (다른 터미널에선 안 보임)

### 5.1 백그라운드 작업 만들기
``` bash
sleep 100 &
```
&를 붙이면 해당 커맨드가 백그라운드 작업으로 실행된다.
명령어 실행 후 [1] 12345 이런 식으로 job 번호와 PID가 표시된다.

### 5.2 작업 목록 확인
``` bash
jobs
```
예시 출력)
``` text
[1]+  Running                 sleep 100 &
[2]-  Stopped                 vim test.txt
[1], [2] : job 번호 (kill이나 fg/bg에서 %1, %2 형태로 사용)
Running, Stopped : 상태
+, - : 기본 포커스 job 표시
```

### 5.3 포어그라운드/백그라운드 전환
``` bash
fg %1   # job 1을 포어그라운드로
bg %2   # job 2를 백그라운드로 재시작
Ctrl + Z # 현재 실행 중인 프로세스를 일시정지(Stopped) 시키고 job으로 보냄
```
그 후 bg나 fg로 다시 이어서 실행 가능

## 6. kill – 프로세스/잡에 시그널 보내기
kill은 단순 “죽이기”가 아니라 프로세스에 시그널을 보내는 명령이다.
기본은 TERM(15) 시그널이고, 필요하면 다른 시그널도 쓸 수 있다.

### 6.1 기본 사용법
``` bash
kill PID
```
기본 시그널: SIGTERM(15) – 정상 종료 요청

프로세스가 말을 안 들으면 -9(SIGKILL)을 사용하지만, 가능하면 마지막 수단으로만.

### 6.2 자주 쓰는 시그널
``` text
1	SIGHUP	설정 다시 읽기(데몬 재시작 느낌)
2	SIGINT	인터럽트 (터미널의 Ctrl + C와 동일)
9	SIGKILL	강제 종료 (잡을 기회 없이 바로 죽음)
15	SIGTERM	정상 종료 요청 (기본값)
18 / 19 등	SIGCONT / SIGSTOP	일시정지/재개
```

시그널 리스트 확인:
``` bash
kill -l
```
### 6.3 예시
예시 1) 정상 종료 요청
``` bash
kill 1234          # SIGTERM (기본)
kill -15 1234      # 명시적으로 SIGTERM
```
예시 2) 강제 종료
``` bash
kill -9 1234
```
주의: -9는 정리할 기회 없이 프로세스를 바로 죽여서
임시 파일, 락 파일 등이 남을 수 있다.
가능하면 15 → 안 되면 9 순서로 쓰는 게 좋다.

예시 3) job 번호로 종료하기
jobs 출력이 다음과 같다고 하자:

``` text
[1]+  Running                 sleep 100 &
```
``` bash
kill %1     # job 번호로 종료
```
